# M11 高级功能模块

## 模块概述

高级功能模块提供历史版本回溯、智能分类、极速秒传、离线下载和后台静默传输等增值功能，提升网盘的使用效率和体验。

## 数据库设计

```sql
-- 文件版本
CREATE TABLE file_versions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    file_id UUID NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    version_number INTEGER NOT NULL,
    storage_key VARCHAR(512) NOT NULL,     -- MinIO 版本对象键
    size BIGINT NOT NULL,
    file_hash VARCHAR(64),
    description VARCHAR(255),              -- 版本说明
    created_by UUID REFERENCES users(id),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    CONSTRAINT unique_file_version UNIQUE (file_id, version_number)
);

-- 秒传指纹索引
CREATE TABLE file_fingerprints (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    file_hash VARCHAR(64) NOT NULL,        -- SHA-256 全文件哈希
    size BIGINT NOT NULL,
    storage_key VARCHAR(512) NOT NULL,      -- 已有文件的 MinIO 键
    reference_count INTEGER DEFAULT 1,     -- 引用计数
    first_uploaded_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    CONSTRAINT unique_file_hash UNIQUE (file_hash, size)
);

-- 离线下载任务
CREATE TABLE offline_downloads (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    url VARCHAR(2048) NOT NULL,
    file_name VARCHAR(255),
    folder_id UUID REFERENCES folders(id),
    status VARCHAR(20) DEFAULT 'pending',  -- pending/downloading/completed/failed
    progress REAL DEFAULT 0,               -- 0-100
    file_size BIGINT,
    storage_key VARCHAR(512),
    error_message TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    completed_at TIMESTAMP WITH TIME ZONE,
    CONSTRAINT fk_offline_download_folder FOREIGN KEY (folder_id) REFERENCES folders(id)
);

-- 智能分类规则
CREATE TABLE classification_rules (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    name VARCHAR(100) NOT NULL,
    condition_type VARCHAR(50) NOT NULL,    -- extension/mime_type/size/name_pattern
    condition_value VARCHAR(255) NOT NULL,
    target_folder_id UUID REFERENCES folders(id),
    is_auto BOOLEAN DEFAULT FALSE,         -- 是否自动执行
    priority INTEGER DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## 子模块详细设计

### M11.1 历史版本

**前端**：
- 文件右键菜单「历史版本」
- 版本列表面板：
  - 版本号、大小、修改时间、修改说明
  - 当前版本标识
  - 操作：预览、下载、回滚到此版本
- 回滚确认：「确定将文件回滚到版本 N？当前版本将保存为新版本」

**后端**：
- `GET /api/v1/files/{id}/versions` — 版本列表
- 响应：
```json
{
  "code": "SUCCESS",
  "data": {
    "current_version": 5,
    "versions": [
      { "version": 5, "size": 1024, "created_at": "...", "description": "" },
      { "version": 4, "size": 980, "created_at": "...", "description": "修复错别字" }
    ]
  }
}
```
- `POST /api/v1/files/{id}/versions/{version}/rollback` — 回滚版本
  - 逻辑：将指定版本的 MinIO 对象复制为新版本，更新 file 记录
- `GET /api/v1/files/{id}/versions/{version}/download` — 下载指定版本
- `DELETE /api/v1/files/{id}/versions/{version}` — 删除指定版本

**版本保留策略**：
- 默认保留最近 30 个版本
- 超出版限自动清理最旧版本
- 可在设置中自定义保留数量

### M11.2 智能分类

**前端**：
- 设置页「智能分类」
- 分类规则列表：条件 → 目标文件夹
- 新建规则：
  - 条件类型：扩展名 / MIME 类型 / 文件大小 / 文件名匹配
  - 条件值：`.jpg,.png` / `image/*` / `>10MB` / `*报告*`
  - 目标文件夹：选择网盘中的文件夹
  - 自动执行开关
- 手动执行分类：扫描所有文件，按规则归类

**后端**：
- `POST /api/v1/classification/rules` — 创建分类规则
- `GET /api/v1/classification/rules` — 规则列表
- `PUT | /api/v1/classification/rules/{id}` — 更新规则
- `DELETE /api/v1/classification/rules/{id}` — 删除规则
- `POST /api/v1/classification/execute` — 执行分类（异步任务）
- `GET /api/v1/classification/suggestions` — 获取分类建议（基于文件类型统计）

### M11.3 极速秒传

**前端**：
- 上传流程中，计算完文件哈希后先调用秒传检测接口
- 如果命中秒传：
  - 进度直接跳到 100%
  - 提示「秒传成功」
  - 不实际上传文件数据

**秒传流程**：
```
1. 前端计算文件 SHA-256
2. 调用 POST /api/v1/files/rapid-upload
3. 后端查询 file_fingerprints 表
4. 命中 → MinIO 复制对象 + 创建 file 记录 → 秒传成功
5. 未命中 → 返回需要上传，走正常上传流程
```

**后端**：
- `POST /api/v1/files/rapid-upload`
```json
{
  "name": "video.mp4",
  "folder_id": "uuid",
  "size": 1073741824,
  "file_hash": "sha256..."
}
```
- 响应（命中）：
```json
{
  "code": "SUCCESS",
  "data": { "file_id": "uuid", "rapid": true }
}
```
- 响应（未命中）：
```json
{
  "code": "RAPID_MISS",
  "message": "文件指纹未命中，请使用普通上传",
  "data": { "rapid": false }
}
```

**指纹管理**：
- 首次上传完成时，写入 file_fingerprints
- 删除文件时，reference_count - 1；为 0 时清理记录和 MinIO 对象
- 哈希计算：前端 Web Worker 分块计算，不阻塞 UI

### M11.4 离线下载

**前端**：
- 工具栏「离线下载」按钮
- 输入下载 URL
- 选择保存到网盘的目标文件夹
- 离线下载列表：URL、状态、进度、操作

**后端**：
- `POST /api/v1/offline-downloads` — 创建离线下载任务
```json
{ "url": "https://example.com/file.zip", "folder_id": "uuid" }
```
- `GET /api/v1/offline-downloads` — 任务列表
- `DELETE /api/v1/offline-downloads/{id}` — 取消/删除任务
- `POST /api/v1/offline-downloads/{id}/pause` — 暂停
- `POST /api/v1/offline-downloads/{id}/resume` — 继续

**后端下载实现**：
- 使用 `httpx` 异步下载
- 下载到临时目录 → 上传到 MinIO → 创建 file 记录
- 支持 HTTP/HTTPS，不支持 BT/磁力链接

### M11.5 后台静默传输

**前端**：
- 设置页「传输」→ 后台传输开关
- 最小化到托盘后继续传输
- 系统通知：传输完成/失败时弹出桌面通知
- 传输完成后自动关机（可选）

**Electron 实现**：
- 窗口关闭时不退出应用，最小化到托盘
- 主进程维持网络连接和传输任务
- 传输状态通过 IPC 同步到渲染进程
- 使用 Electron Notification API 推送桌面通知

## 错误码

| 错误码 | HTTP状态 | 说明 |
|--------|----------|------|
| VERSION_NOT_FOUND | 404 | 版本不存在 |
| CANNOT_DELETE_CURRENT_VERSION | 400 | 不能删除当前版本 |
| RAPID_MISS | 200 | 秒传未命中 |
| INVALID_DOWNLOAD_URL | 400 | 无效的下载链接 |
| DOWNLOAD_FAILED | 500 | 下载失败 |
| CLASSIFICATION_RULE_EXISTS | 409 | 规则已存在 |

## 测试要求

- 覆盖率目标：≥ 94%（复杂任务）
- 关键测试场景：
  - 文件版本创建和回滚
  - 版本保留数量限制
  - 秒传命中和未命中
  - 秒传指纹引用计数
  - 离线下载完整流程
  - 分类规则执行准确性
  - 后台传输不中断
