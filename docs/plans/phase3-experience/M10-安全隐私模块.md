# M10 安全隐私模块

## 模块概述

安全隐私模块为网盘提供多层安全保障，包括二次验证、私密文件夹、端侧加密、设备管理、异地登录提醒和操作日志。确保用户数据和账号的安全性。

## 技术选型

| 层面 | 技术 |
|------|------|
| 二次验证 | TOTP（pyotp 库，兼容 Google Authenticator） |
| 文件加密 | AES-256-GCM（前端加密，后端仅存储密文） |
| 密钥派生 | PBKDF2-SHA256（基于用户密码派生加密密钥） |
| 设备指纹 | User-Agent + 屏幕分辨率 + 时区组合 |
| 操作日志 | 异步写入，不阻塞业务请求 |

CREATE INDEX idx_operation_logs_user ON operation_logs(user_id, created_at DESC);

## 子模块详细设计

### M10.1 二次验证（TOTP）

**前端**：
- 设置页「安全」→ 二次验证开关
- 开启流程：
  1. 生成 TOTP 密钥 + 二维码
  2. 用户用 Google Authenticator 扫码
  3. 输入验证码确认绑定
  4. 展示备用恢复码（10 个，仅显示一次）
- 登录时二次验证页面：输入 6 位验证码
- 支持备用恢复码登录

**后端**：
- `POST /api/v1/security/totp/setup` — 初始化 TOTP
- 响应：
```json
{
  "code": "SUCCESS",
  "data": {
    "secret": "JBSWY3DPEHPK3PXP",
    "qr_code_url": "otpauth://totp/...",
    "backup_codes": ["abc123", "def456", ...]
  }
}
```
- `POST /api/v1/security/totp/verify` — 验证 TOTP 码
- `POST /api/v1/security/totp/enable` — 确认启用
- `DELETE /api/v1/security/totp` — 关闭二次验证（需验证密码）
- `POST /api/v1/auth/login/2fa` — 二次验证登录

### M10.2 私密文件夹

**前端**：
- 文件夹右键菜单「设为私密」
- 设置私密密码
- 访问私密文件夹时弹出密码输入框
- 自动锁定：N 分钟无操作后自动锁定
- 私密文件夹图标特殊标识（锁图标）

**后端**：
- `POST /api/v1/security/private-folders`
```json
{ "folder_id": "uuid", "password": "string", "auto_lock_minutes": 5 }
```
- `POST /api/v1/security/private-folders/{id}/unlock`
```json
{ "password": "string" }
```
- 响应返回临时访问令牌（有效期 = auto_lock_minutes）
- `POST /api/v1/security/private-folders/{id}/lock` — 手动锁定
- `DELETE /api/v1/security/private-folders/{id}` — 取消私密

### M10.3 端侧加密

**前端**：
- 设置页「安全」→ 端侧加密开关
- 开启后上传文件前在本地加密：
  1. 用户输入加密密码
  2. PBKDF2 派生 AES-256 密钥
  3. 文件分块后 AES-256-GCM 加密
  4. 上传密文到服务器
- 下载后在本地解密
- 服务器永远不接触明文和密钥

**加密流程**：
```typescript
async function encryptFile(file: File, password: string): Promise<Blob> {
  // 1. 从密码派生密钥
  const keyMaterial = await crypto.subtle.importKey(
    'raw', new TextEncoder().encode(password), 'PBKDF2', false, ['deriveKey']
  )
  const salt = crypto.getRandomValues(new Uint8Array(16))
  const key = await crypto.subtle.deriveKey(
    { name: 'PBKDF2', salt, iterations: 100000, hash: 'SHA-256' },
    keyMaterial, { name: 'AES-GCM', length: 256 }, false, ['encrypt']
  )
  
  // 2. 分块加密
  const iv = crypto.getRandomValues(new Uint8Array(12))
  const encrypted = await crypto.subtle.encrypt(
    { name: 'AES-GCM', iv }, key, await file.arrayBuffer()
  )
  
  // 3. 拼接 salt + iv + 密文
  return new Blob([salt, iv, encrypted])
}
```

**后端**：
- 无需特殊处理，仅存储密文
- 文件元数据标记 `is_encrypted = TRUE`
- 加密文件不支持在线预览（无法服务端解密）

### M10.4 设备管理

**前端**：
- 设置页「安全」→ 设备管理
- 设备列表：设备名称、类型、位置、最后活跃时间
- 当前设备标识
- 操作：信任设备、远程下线、重命名

**后端**：
- `GET /api/v1/security/devices` — 设备列表
- `PUT /api/v1/security/devices/{id}` — 更新设备信息（重命名/信任）
- `DELETE /api/v1/security/devices/{id}` — 远程下线（撤销该设备所有 Token）

### M10.5 登录保护

**前端**：
- 登录时自动上报设备信息
- 新设备/异地登录收到通知弹窗
- 通知内容：设备类型、IP、位置、时间

**后端**：
- 登录 API 中自动检测：
  1. 设备指纹是否已记录
  2. IP 归属地是否与历史一致
  3. 新设备或异地 → 创建 login_alerts 记录
  4. WebSocket 推送通知给在线客户端
- `GET /api/v1/security/alerts` — 获取登录提醒
- `PUT /api/v1/security/alerts/{id}/read` — 标记已读

### M10.6 操作日志

**前端**：
- 设置页「安全」→ 操作日志
- 按时间倒序展示
- 筛选：操作类型、时间范围
- 每条日志显示：操作、对象、时间、IP

**后端**：
- `GET /api/v1/security/logs?action=login&date_from=&date_to=&page=1`
- 日志写入使用异步方式，不阻塞业务请求
- 日志保留策略：90 天，超期自动归档

## 错误码

| 错误码 | HTTP状态 | 说明 |
|--------|----------|------|
| TOTP_NOT_ENABLED | 400 | 未启用二次验证 |
| TOTP_CODE_INVALID | 401 | 验证码错误 |
| TOTP_ALREADY_ENABLED | 409 | 已启用二次验证 |
| PRIVATE_FOLDER_PASSWORD_INVALID | 401 | 私密文件夹密码错误 |
| PRIVATE_FOLDER_LOCKED | 423 | 私密文件夹已锁定 |
| DEVICE_NOT_FOUND | 404 | 设备不存在 |
| CANNOT_LOGOUT_CURRENT | 400 | 不能远程下线当前设备 |

## 测试要求

- 覆盖率目标：≥ 94%（复杂任务）
- 关键测试场景：
  - TOTP 绑定/验证/解绑全流程
  - 备用恢复码登录
  - 私密文件夹锁定/解锁/自动锁定
  - 端侧加密文件上传下载
  - 设备远程下线后 Token 失效
  - 异地登录检测和通知
  - 操作日志完整性
