# M9 标签与批量模块

## 模块概述

标签与批量模块为文件管理提供分类标注和批量操作能力，提升大量文件的管理效率。标签系统支持自定义标签和颜色，批量操作支持多选后的统一处理。

## 子模块详细设计

### M9.1 文件标签

**前端**：
- 标签管理面板：
  - 标签列表（名称 + 颜色色块）
  - 新建标签：名称 + 颜色选择器
  - 编辑标签 / 删除标签
- 文件标签操作：
  - 文件右键菜单「添加标签」
  - 文件详情面板展示已有标签（可移除）
  - 侧边栏标签筛选器：点击标签过滤文件

**后端**：
- `POST /api/v1/tags` — 创建标签
```json
{ "name": "工作", "color": "#FF5733" }
```
- `GET /api/v1/tags` — 获取用户所有标签
- `PUT /api/v1/tags/{id}` — 更新标签
- `DELETE /api/v1/tags/{id}` — 删除标签（同时解除文件关联）
- `POST /api/v1/files/{id}/tags` — 为文件添加标签
```json
{ "tag_ids": ["uuid1", "uuid2"] }
```
- `DELETE /api/v1/files/{id}/tags/{tag_id}` — 移除文件标签
- `GET /api/v1/files?tag_id={id}` — 按标签筛选文件

**标签使用统计**：
- `GET /api/v1/tags/stats` — 各标签下文件数量
```json
{
  "code": "SUCCESS",
  "data": [
    { "tag_id": "uuid", "name": "工作", "color": "#FF5733", "file_count": 23 },
    { "tag_id": "uuid", "name": "重要", "color": "#E74C3C", "file_count": 8 }
  ]
}
```

### M9.2 批量操作

**前端**：
- 多选方式：
  - Ctrl + 点击：追加选中
  - Shift + 点击：范围选中
  - 表头全选复选框：当前页全选
  - 快捷键 Ctrl+A：全选
- 选中后操作栏（浮动在底部）：
  - 批量移动
  - 批量复制
  - 批量删除
  - 批量添加标签
  - 批量下载（打包为 ZIP）
  - 取消选择
- 选中数量实时显示

**后端**：
- 批量接口统一接受 ID 数组
- `POST /api/v1/files/batch/move`
```json
{ "file_ids": ["uuid1", "uuid2"], "folder_ids": ["uuid3"], "target_folder_id": "uuid" }
```
- `POST /api/v1/files/batch/copy`
```json
{ "file_ids": ["uuid1", "uuid2"], "folder_ids": ["uuid3"], "target_folder_id": "uuid" }
```
- `POST /api/v1/files/batch/delete`
```json
{ "file_ids": ["uuid1"], "folder_ids": ["uuid2", "uuid3"] }
```
- `POST /api/v1/files/batch/tags`
```json
{ "file_ids": ["uuid1", "uuid2"], "tag_ids": ["uuid4"] }
```
- `POST /api/v1/files/batch/download` — 批量下载打包
```json
{ "file_ids": ["uuid1", "uuid2"], "folder_ids": ["uuid3"] }
```
- 响应：
```json
{
  "code": "SUCCESS",
  "data": {
    "task_id": "uuid",       // 异步打包任务ID
    "status": "packing",     // packing/ready/failed
    "estimated_files": 5
  }
}
```
- `GET /api/v1/files/batch/download/{task_id}/status` — 查询打包状态
- `GET /api/v1/files/batch/download/{task_id}` — 下载打包文件

**批量打包逻辑**：
1. 创建打包任务记录
2. 后台异步将文件打包为 ZIP
3. ZIP 上传到 MinIO 临时存储
4. 更新任务状态为 ready
5. 返回下载签名 URL
6. 24小时后自动清理临时 ZIP

**数据库设计**：
```sql
CREATE TABLE batch_download_tasks (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    storage_key VARCHAR(512),
    file_count INTEGER,
    total_size BIGINT,
    status VARCHAR(20) DEFAULT 'packing',  -- packing/ready/failed
    error_message TEXT,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    expires_at TIMESTAMP WITH TIME ZONE DEFAULT (NOW() + INTERVAL '24 hours')
);
```

## 错误码

| 错误码 | HTTP状态 | 说明 |
|--------|----------|------|
| TAG_NOT_FOUND | 404 | 标签不存在 |
| TAG_NAME_EXISTS | 409 | 标签名称已存在 |
| BATCH_TASK_NOT_FOUND | 404 | 批量任务不存在 |
| BATCH_TASK_EXPIRED | 410 | 批量任务已过期 |
| BATCH_SIZE_EXCEEDED | 400 | 批量操作数量超限（最多500个） |

## 测试要求

- 覆盖率目标：≥ 91%（中等任务）
- 关键测试场景：
  - 标签 CRUD 完整流程
  - 文件关联/取消关联标签
  - 标签筛选文件
  - 批量移动/复制/删除
  - 批量下载打包和过期清理
  - 并发批量操作一致性
