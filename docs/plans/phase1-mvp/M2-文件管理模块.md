# M2 文件管理模块

## 模块概述

文件管理模块是网盘的核心功能，负责文件和文件夹的增删改查、目录导航、排序筛选和搜索。采用虚拟文件树模型，文件夹为逻辑容器，文件存储在 MinIO 对象存储中。

## 核心数据模型

```sql
CREATE TABLE folders (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    parent_id UUID REFERENCES folders(id) ON DELETE CASCADE,
    name VARCHAR(255) NOT NULL,
    path VARCHAR(1024) NOT NULL,       -- 物理路径：/root/docs/project
    is_deleted BOOLEAN DEFAULT FALSE,
    deleted_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    CONSTRAINT unique_folder_name UNIQUE (user_id, parent_id, name, is_deleted)
);

CREATE TABLE files (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    folder_id UUID REFERENCES folders(id) ON DELETE SET NULL,
    name VARCHAR(255) NOT NULL,
    extension VARCHAR(50),              -- 文件扩展名
    mime_type VARCHAR(255),
    size BIGINT NOT NULL DEFAULT 0,     -- 字节数
    storage_key VARCHAR(512) NOT NULL,  -- MinIO 对象键
    file_hash VARCHAR(64),              -- SHA-256 用于秒传比对
    is_deleted BOOLEAN DEFAULT FALSE,
    deleted_at TIMESTAMP WITH TIME ZONE,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);

CREATE TABLE file_tags (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    name VARCHAR(50) NOT NULL,
    color VARCHAR(7),                   -- 十六进制颜色 #FF5733
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    CONSTRAINT unique_tag_name UNIQUE (user_id, name)
);

CREATE TABLE file_tag_relations (
    file_id UUID NOT NULL REFERENCES files(id) ON DELETE CASCADE,
    tag_id UUID NOT NULL REFERENCES file_tags(id) ON DELETE CASCADE,
    PRIMARY KEY (file_id, tag_id)
);
```

## 子模块详细设计

### M2.1 文件列表

**前端**：
- 路由：`/files`（主页）
- 双视图切换：表格视图 / 网格视图
- 表格列：名称、大小、类型、修改时间、操作按钮
- 网格卡片：缩略图/图标 + 文件名 + 大小
- 支持分页加载（每页 50 项）
- 空文件夹友好提示

**后端**：
- `GET /api/v1/files?folder_id={id}&page=1&page_size=50&sort=name&order=asc`
- 响应体：
```json
{
  "code": "SUCCESS",
  "message": "获取成功",
  "data": {
    "folders": [...],
    "files": [...],
    "total": 100,
    "page": 1,
    "page_size": 50
  }
}
```
- 逻辑：优先返回子文件夹，再返回文件

### M2.2 文件夹导航

**前端**：
- 面包屑导航：`全部文件 > 文档 > 项目`
- 左侧目录树（懒加载展开）
- 点击文件夹加载子目录内容

**后端**：
- `GET /api/v1/folders/tree` — 获取完整目录树
- `GET /api/v1/folders/{id}/path` — 获取面包屑路径链

### M2.3 新建

**前端**：
- 工具栏「新建」按钮 → 下拉菜单：新建文件夹、新建文本文件
- 新建文件夹弹窗：输入名称
- 新建后自动进入重命名状态

**后端**：
- `POST /api/v1/folders` — 创建文件夹
```json
{ "parent_id": "uuid|null", "name": "新建文件夹" }
```
- `POST /api/v1/files` — 创建空文件
```json
{ "folder_id": "uuid", "name": "未命名.txt" }
```

### M2.4 重命名

**前端**：
- 右键菜单「重命名」/ 双击文件名
- 行内编辑模式，Enter 确认，Esc 取消
- 校验：名称不能为空、不能含特殊字符 `\/:*?"<>|`

**后端**：
- `PUT /api/v1/files/{id}/rename`
```json
{ "name": "新名称.txt" }
```
- `PUT /api/v1/folders/{id}/rename`
```json
{ "name": "新名称" }
```
- 逻辑：检查同级目录下同名冲突，更新 path 字段（文件夹需递归更新子路径）

### M2.5 移动/复制

**前端**：
- 右键菜单「移动到」/「复制到」
- 弹出目录选择器（树形），支持新建文件夹
- 移动时目标目录不能是自身子目录

**后端**：
- `POST /api/v1/files/move`
```json
{ "file_ids": ["uuid1", "uuid2"], "target_folder_id": "uuid" }
```
- `POST /api/v1/files/copy`
```json
{ "file_ids": ["uuid1", "uuid2"], "target_folder_id": "uuid" }
```
- 复制逻辑：在 MinIO 中复制对象，创建新 file 记录
- 移动逻辑：更新 folder_id，不移动 MinIO 对象（节省 IO）

### M2.6 删除与回收站

**前端**：
- 删除确认弹窗：「确定将 N 个文件移至回收站？」
- 回收站页面：列出已删除文件，显示原始位置和删除时间
- 操作：恢复（回到原位置）、彻底删除
- 清空回收站按钮

**后端**：
- `DELETE /api/v1/files` — 软删除
```json
{ "file_ids": ["uuid1"], "folder_ids": ["uuid2"] }
```
- `GET /api/v1/trash` — 回收站列表
- `POST /api/v1/trash/restore` — 恢复
```json
{ "items": [{"id": "uuid", "type": "file|folder"}] }
```
- `DELETE /api/v1/trash/purge` — 彻底删除（同时删除 MinIO 对象）
- `DELETE /api/v1/trash/empty` — 清空回收站
- 软删除逻辑：`is_deleted = TRUE, deleted_at = NOW()`
- 30天自动清理定时任务

### M2.7 排序与筛选

**前端**：
- 表头点击排序：名称、大小、类型、修改时间
- 筛选栏：文件类型（图片/文档/视频/音频/其他）、时间范围

**后端**：
- 排序参数：`sort=name|size|type|created_at&order=asc|desc`
- 筛选参数：`type=image|document|video|audio&date_from=&date_to=`

### M2.8 搜索

**前端**：
- 顶部搜索栏，输入即搜（debounce 300ms）
- 搜索结果页：高亮匹配关键词，显示文件路径
- 支持搜索文件名

**后端**：
- `GET /api/v1/files/search?keyword=xxx&type=image&limit=50`
- 使用 PostgreSQL `ILIKE` 实现模糊匹配
- 返回匹配的文件和文件夹列表

## API 汇总

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/files | 文件列表 |
| POST | /api/v1/files | 创建空文件 |
| PUT | /api/v1/files/{id}/rename | 重命名文件 |
| POST | /api/v1/files/move | 移动文件 |
| POST | /api/v1/files/copy | 复制文件 |
| DELETE | /api/v1/files | 删除文件（软删除） |
| GET | /api/v1/files/search | 搜索文件 |
| GET | /api/v1/folders/tree | 目录树 |
| POST | /api/v1/folders | 创建文件夹 |
| PUT | /api/v1/folders/{id}/rename | 重命名文件夹 |
| GET | /api/v1/folders/{id}/path | 获取路径链 |
| GET | /api/v1/trash | 回收站列表 |
| POST | /api/v1/trash/restore | 恢复文件 |
| DELETE | /api/v1/trash/purge | 彻底删除 |
| DELETE | /api/v1/trash/empty | 清空回收站 |

## 错误码

| 错误码 | HTTP状态 | 说明 |
|--------|----------|------|
| FOLDER_NOT_FOUND | 404 | 文件夹不存在 |
| FILE_NOT_FOUND | 404 | 文件不存在 |
| NAME_CONFLICT | 409 | 同级存在同名文件/文件夹 |
| INVALID_NAME | 400 | 文件名包含非法字符 |
| CIRCULAR_MOVE | 400 | 不能移动到自身子目录 |
| STORAGE_EXCEEDED | 403 | 存储空间不足 |

## 测试要求

- 覆盖率目标：≥ 94%（复杂任务）
- 关键测试场景：
  - 文件CRUD全流程
  - 多层嵌套文件夹的路径更新
  - 重命名同级冲突检测
  - 移动循环引用检测
  - 软删除与恢复一致性
  - 回收站30天自动清理
  - 搜索模糊匹配准确性
