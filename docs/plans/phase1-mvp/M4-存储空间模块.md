# M4 存储空间模块

## 模块概述

存储空间模块负责用户容量统计、配额管理和容量预警。为上传操作提供前置校验，防止用户超出存储配额。

## 子模块详细设计

### M4.1 容量展示

**前端**：
- 侧边栏底部或设置页面展示容量信息
- 环形进度图：已用/总量
- 文字信息：`已用 2.3GB / 共 5GB`
- 分类统计：图片、视频、文档、其他各占多少

**后端**：
- `GET /api/v1/storage/usage`
- 响应体：
```json
{
  "code": "SUCCESS",
  "data": {
    "total_quota": 5368709120,
    "used_quota": 2432696320,
    "available_quota": 2936012800,
    "usage_percentage": 45.3,
    "category_breakdown": {
      "image": 838860800,
      "video": 1073741824,
      "document": 209715200,
      "audio": 104857600,
      "other": 205520896
    }
  }
}
```
- 分类统计逻辑：根据 mime_type 聚合 SUM(size)

### M4.2 配额管理

**前端**：
- 上传前校验：文件大小 > 剩余空间 → 提示"存储空间不足"
- 容量预警：
  - 已用 > 80% → 黄色提示
  - 已用 > 95% → 红色警告
- 设置页展示配额信息

**后端**：
- 配额检查中间件：在文件上传前拦截
- 逻辑：
  1. 查询用户 storage_quota 和 storage_used
  2. 判断 `storage_used + file_size > storage_quota`
  3. 超出则返回 `STORAGE_EXCEEDED` 错误
- storage_used 更新时机：
  - 文件上传完成 → 增加
  - 文件彻底删除 → 减少
  - 使用 PostgreSQL 行级锁防止并发更新不一致

```sql
-- 原子更新存储用量
UPDATE users
SET storage_used = storage_used + :delta
WHERE id = :user_id AND storage_used + :delta <= storage_quota
RETURNING storage_used;
```

## API 汇总

| 方法 | 路径 | 说明 |
|------|------|------|
| GET | /api/v1/storage/usage | 获取存储使用情况 |

## 错误码

| 错误码 | HTTP状态 | 说明 |
|--------|----------|------|
| STORAGE_EXCEEDED | 403 | 存储空间不足 |

## 测试要求

- 覆盖率目标：≥ 91%（中等任务）
- 关键测试场景：
  - 容量统计准确性
  - 分类统计聚合正确性
  - 配额检查拦截
  - 并发上传时 storage_used 一致性
  - 删除文件后容量回收
