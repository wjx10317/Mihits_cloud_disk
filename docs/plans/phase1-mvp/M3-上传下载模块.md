# M3 上传下载模块

## 模块概述

上传下载模块是网盘的核心传输能力，支持普通上传、大文件分片上传、断点续传、下载管理以及传输队列控制。大文件分片上传基于 MinIO Multipart Upload API 实现。

## 技术选型

| 层面 | 技术 |
|------|------|
| 分片上传 | MinIO Multipart Upload API |
| 分片大小 | 5MB（MinIO 最小分片） |
| 最大文件 | 5TB（MinIO 限制） |
| 最大分片数 | 10000 |
| 并发上传 | asyncio.Semaphore 控制并发 |
| 传输队列 | Pinia Store + localStorage 持久化 |
| 断点续传 | 分片状态持久化 + 服务端查询 |

## 子模块详细设计

### M3.1 拖拽上传

**前端**：
- 文件管理页面中央拖拽区域
- 拖入时高亮显示上传区域
- 拖拽释放触发上传流程
- 支持拖入文件和文件夹
- 文件夹上传：使用 webkitdirectory 属性或递归读取

**后端**：
- `POST /api/v1/files/upload` — 普通上传（小文件 ≤ 5MB）
- Content-Type: `multipart/form-data`
- 请求体：file + folder_id
- 逻辑：
  1. 校验存储空间是否足够
  2. 生成唯一 storage_key（`{user_id}/{uuid}.{ext}`）
  3. 上传到 MinIO
  4. 创建 file 记录
  5. 更新用户 storage_used

### M3.2 大文件分片上传

**前端流程**：
1. 选择文件 → 计算 SHA-256 哈希（Web Worker，分块计算）
2. 调用初始化接口获取 upload_id
3. 将文件切割为 5MB 分片
4. 并行上传分片（最多 3 个并发）
5. 全部分片上传完成后调用合并接口
6. 上传进度 = 已完成分片数 / 总分片数

**后端接口**：

1. **初始化分片上传**
- `POST /api/v1/files/upload/init`
```json
{
  "name": "video.mp4",
  "folder_id": "uuid",
  "size": 1073741824,
  "file_hash": "sha256...",
  "total_chunks": 200
}
```
- 响应：
```json
{
  "code": "SUCCESS",
  "data": {
    "upload_id": "string",
    "storage_key": "string",
    "chunk_size": 5242880
  }
}
```

2. **上传分片**
- `PUT /api/v1/files/upload/chunk`
- 请求体：`multipart/form-data`（chunk + upload_id + chunk_number）
- 幂等设计：相同 upload_id + chunk_number 可重复上传

3. **合并分片**
- `POST /api/v1/files/upload/complete`
```json
{
  "upload_id": "string",
  "name": "video.mp4",
  "folder_id": "uuid",
  "size": 1073741824,
  "file_hash": "sha256..."
}
```
- 逻辑：调用 MinIO CompleteMultipartUpload，创建 file 记录

### M3.3 断点续传

**前端**：
- 上传任务持久化到 IndexedDB（upload_tasks 表）
- 记录：upload_id、文件信息、已完成分片列表
- 应用重启时扫描未完成任务，查询服务端分片状态后继续

**后端**：
- `GET /api/v1/files/upload/{upload_id}/chunks` — 查询已上传分片列表
- 响应：
```json
{
  "code": "SUCCESS",
  "data": {
    "upload_id": "string",
    "completed_chunks": [1, 2, 3, 5, 8],
    "total_chunks": 200,
    "status": "uploading|completed|expired"
  }
}
```
- 上传 ID 有效期：24小时（过期自动清理 MinIO 分片）

### M3.4 下载

**前端**：
- 文件右键菜单「下载」
- 下载进度条
- 大文件分片下载（可选）
- 下载保存路径选择对话框

**后端**：
- `GET /api/v1/files/{id}/download`
- 逻辑：
  1. 校验文件权限
  2. 生成 MinIO 预签名 URL（有效期 1h）
  3. 返回签名 URL，前端直接从 MinIO 下载
- 响应：
```json
{
  "code": "SUCCESS",
  "data": {
    "download_url": "https://minio.../bucket/key?signature...",
    "expires_in": 3600
  }
}
```

### M3.5 传输管理

**前端**：
- 底部传输面板（可展开/收起）
- 分 Tab：上传队列 / 下载队列
- 每个任务显示：文件名、进度、速度、剩余时间
- 操作按钮：暂停、继续、取消、重试

**Pinia Store 设计**：
```typescript
interface TransferTask {
  id: string
  type: 'upload' | 'download'
  fileName: string
  fileSize: number
  progress: number        // 0-100
  speed: number           // bytes/s
  status: 'pending' | 'active' | 'paused' | 'completed' | 'failed' | 'cancelled'
  error?: string
  createdAt: number
  completedAt?: number
  // 分片上传额外字段
  uploadId?: string
  totalChunks?: number
  completedChunks?: number[]
}
```

### M3.6 失败重试

**前端**：
- 单个分片上传失败自动重试，最多 3 次
- 3 次仍失败则标记任务为 failed
- 用户可手动点击重试按钮
- 重试时从失败分片继续，不重新上传已成功分片

**后端**：
- 分片上传接口幂等设计：相同 upload_id + chunk_number 覆盖写入
- 合并接口校验所有分片完整性

## 数据库设计

```sql
CREATE TABLE upload_sessions (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    upload_id VARCHAR(255) NOT NULL,     -- MinIO Upload ID
    storage_key VARCHAR(512) NOT NULL,
    file_name VARCHAR(255) NOT NULL,
    folder_id UUID REFERENCES folders(id),
    total_chunks INTEGER NOT NULL,
    file_size BIGINT NOT NULL,
    file_hash VARCHAR(64),
    status VARCHAR(20) DEFAULT 'uploading',  -- uploading/completed/expired
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    expires_at TIMESTAMP WITH TIME ZONE DEFAULT (NOW() + INTERVAL '24 hours')
);

CREATE INDEX idx_upload_sessions_user ON upload_sessions(user_id);
```

## API 汇总

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/v1/files/upload | 普通上传（小文件） |
| POST | /api/v1/files/upload/init | 初始化分片上传 |
| PUT | /api/v1/files/upload/chunk | 上传分片 |
| POST | /api/v1/files/upload/complete | 合并分片 |
| GET | /api/v1/files/upload/{upload_id}/chunks | 查询已上传分片 |
| GET | /api/v1/files/{id}/download | 获取下载链接 |

## 错误码

| 错误码 | HTTP状态 | 说明 |
|--------|----------|------|
| UPLOAD_SESSION_NOT_FOUND | 404 | 上传会话不存在 |
| UPLOAD_SESSION_EXPIRED | 410 | 上传会话已过期 |
| CHUNK_OUT_OF_RANGE | 400 | 分片编号超出范围 |
| STORAGE_EXCEEDED | 403 | 存储空间不足 |
| FILE_HASH_MISMATCH | 400 | 文件哈希校验失败 |
| UPLOAD_CONFLICT | 409 | 同名文件已存在 |

## 测试要求

- 覆盖率目标：≥ 94%（复杂任务）
- 关键测试场景：
  - 小文件上传完整流程
  - 大文件分片上传完整流程
  - 分片上传中断后续传
  - 上传过期会话处理
  - 存储空间不足拦截
  - 文件哈希校验
  - 下载签名URL生成
  - 并发上传控制
  - 失败重试机制
