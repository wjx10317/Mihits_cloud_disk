# M1 用户认证模块

## 模块概述

用户认证模块是网盘客户端的入口，负责用户注册、登录和会话管理。采用 JWT 双 Token 机制（Access Token + Refresh Token），确保安全性和用户体验。

## 技术选型

| 层面 | 技术 |
|------|------|
| 密码加密 | bcrypt（cost factor=12） |
| Token 签发 | PyJWT + RS256 算法 |
| Token 存储 | electron-store（加密存储） |
| 邮箱校验 | pydantic EmailStr |
| 限流 | slowapi（注册/登录接口限流） |

## 子模块详细设计

### M1.1 用户注册

**前端**：
- 路由：`/register`
- 表单字段：邮箱、用户名、密码、确认密码
- 校验规则：
  - 邮箱：格式校验
  - 用户名：3-20字符，仅字母数字下划线
  - 密码：8-32字符，至少含大小写字母和数字
  - 确认密码：与密码一致
- 提交后自动跳转登录页

**后端**：
- `POST /api/v1/auth/register`
- 请求体：
```json
{
  "email": "user@example.com",
  "username": "string",
  "password": "string"
}
```
- 响应体：
```json
{
  "code": "SUCCESS",
  "message": "注册成功"
}
```
- 逻辑：
  1. 校验邮箱/用户名唯一性
  2. bcrypt 加密密码
  3. 写入 users 表
  4. 初始化用户存储配额（默认 5GB）
  5. 在 MinIO 创建用户 bucket

**数据库表**：
```sql
CREATE TABLE users (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    email VARCHAR(255) UNIQUE NOT NULL,
    username VARCHAR(50) UNIQUE NOT NULL,
    password_hash VARCHAR(255) NOT NULL,
    storage_quota BIGINT NOT NULL DEFAULT 5368709120,  -- 5GB
    storage_used BIGINT NOT NULL DEFAULT 0,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    updated_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    is_active BOOLEAN DEFAULT TRUE
);
```

### M1.2 用户登录

**前端**：
- 路由：`/login`
- 表单字段：邮箱、密码
- 登录成功后：
  1. 存储 Access Token 和 Refresh Token
  2. 跳转到文件管理主页

**后端**：
- `POST /api/v1/auth/login`
- 请求体：
```json
{
  "email": "user@example.com",
  "password": "string"
}
```
- 响应体：
```json
{
  "code": "SUCCESS",
  "message": "登录成功",
  "data": {
    "access_token": "string",
    "refresh_token": "string",
    "token_type": "Bearer",
    "expires_in": 3600,
    "user": {
      "id": "uuid",
      "email": "user@example.com",
      "username": "string",
      "storage_quota": 5368709120,
      "storage_used": 0
    }
  }
}
```
- 逻辑：
  1. 查询用户，校验密码
  2. 签发 Access Token（有效期 1h）和 Refresh Token（有效期 7d）
  3. 记录登录设备信息

**限流**：同一 IP 每分钟最多 5 次登录请求

### M1.3 会话管理

**前端**：
- Token 存储在 electron-store（加密）
- Axios 请求拦截器：自动附加 Authorization Header
- Axios 响应拦截器：
  - 401 且 Token 过期 → 自动调用刷新接口
  - 刷新失败 → 跳转登录页
- 应用启动时检查 Token 有效性

**后端**：
- `POST /api/v1/auth/refresh`
- 请求体：`{ "refresh_token": "string" }`
- 响应体：新的 Access Token + Refresh Token 对
- Token 黑名单（Redis 或数据库，用于主动登出）

**数据库表**：
```sql
CREATE TABLE refresh_tokens (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    token_hash VARCHAR(255) NOT NULL,
    device_info VARCHAR(255),
    expires_at TIMESTAMP WITH TIME ZONE NOT NULL,
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    is_revoked BOOLEAN DEFAULT FALSE
);

CREATE TABLE login_devices (
    id UUID PRIMARY KEY DEFAULT gen_random_uuid(),
    user_id UUID NOT NULL REFERENCES users(id) ON DELETE CASCADE,
    device_name VARCHAR(255),
    device_type VARCHAR(50),
    ip_address VARCHAR(45),
    last_active_at TIMESTAMP WITH TIME ZONE DEFAULT NOW(),
    created_at TIMESTAMP WITH TIME ZONE DEFAULT NOW()
);
```

## API 汇总

| 方法 | 路径 | 说明 |
|------|------|------|
| POST | /api/v1/auth/register | 用户注册 |
| POST | /api/v1/auth/login | 用户登录 |
| POST | /api/v1/auth/refresh | 刷新Token |
| POST | /api/v1/auth/logout | 退出登录（撤销Token） |

## 错误码

| 错误码 | HTTP状态 | 说明 |
|--------|----------|------|
| EMAIL_ALREADY_EXISTS | 409 | 邮箱已注册 |
| USERNAME_ALREADY_EXISTS | 409 | 用户名已存在 |
| INVALID_CREDENTIALS | 401 | 邮箱或密码错误 |
| TOKEN_EXPIRED | 401 | Token已过期 |
| TOKEN_INVALID | 401 | Token无效 |
| TOO_MANY_REQUESTS | 429 | 请求过于频繁 |

## 测试要求

- 覆盖率目标：≥ 91%（中等任务）
- 关键测试场景：
  - 注册：正常注册、重复邮箱、重复用户名、弱密码
  - 登录：正常登录、错误密码、不存在用户、限流
  - Token：正常刷新、过期刷新、黑名单Token、并发刷新
